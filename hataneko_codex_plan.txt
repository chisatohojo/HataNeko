破綻ねこ 開発企画書 / Codex向け実装指示書
============================================================

アプリ名:
  破綻ねこ

サブタイトル:
  生成画像の破綻候補を見つける検品アプリ

目的:
  Stable Diffusion等で大量生成した画像を、ローカル環境で素早く確認・仕分けできるWindows向けデスクトップアプリを作る。
  本アプリはAIイラストの破綻を100%自動判定するものではなく、指・腕・頭・顔・解像度・読み込みエラーなど、明確に怪しい候補を抽出して、人間の検品作業を楽にするための補助ツールである。

重要方針:
  - 初期版では高度なAI破綻判定は不要。
  - まずは画像ビューア、仕分け、削除、簡易スキャンを完成させる。
  - 後から検出器を追加できるよう、detectors/配下に疎結合な検出モジュールを追加できる構成にする。
  - 「破綻」と断定せず、「破綻候補」「要確認」として表示する。
  - 削除は完全削除ではなく、安全な退避を基本とする。

------------------------------------------------------------
1. 想定ユーザー
------------------------------------------------------------

- Stable Diffusion等で画像を大量生成するユーザー
- 投稿前に破綻画像を取り除きたいユーザー
- BOOTH、Patreon、Fantia、SNS投稿前に画像検品したいユーザー
- 生成画像を OK / 修正 / NG / 削除候補 に素早く仕分けしたいユーザー

------------------------------------------------------------
2. 対象OS
------------------------------------------------------------

- Windows 10
- Windows 11

初期版はWindows向けローカルアプリとして作成する。

4. MVPで実装する機能
------------------------------------------------------------

4.1 フォルダ読み込み

ユーザーが任意の画像フォルダを選択できるようにする。

対応拡張子:
  - .png
  - .jpg
  - .jpeg
  - .webp
  - .bmp

フォルダ選択後、対象画像一覧を読み込み、サムネイルとプレビューを表示する。


4.2 画像ビューア

現在選択中の画像を大きく表示する。

必要な表示情報:
  - ファイル名
  - 現在位置 例: 12 / 240
  - 解像度 例: 1024 x 1536
  - ファイルサイズ
  - スキャン結果ステータス

必要な表示機能:
  - ウィンドウサイズに合わせて画像全体表示
  - 前の画像 / 次の画像へ移動
  - 画像上に警告マーカーや赤枠を表示できる土台


4.3 サムネイル一覧

左側に画像サムネイル一覧を表示する。

各サムネイルに表示する情報:
  - 小さなプレビュー
  - ファイル名
  - ステータス

ステータス候補:
  - 未確認
  - Suspicious
  - OK
  - Fix
  - NG
  - Deleted


4.4 仕分け機能

画像を以下のフォルダへ移動できるようにする。
入力フォルダと同階層に以下を自動作成する。

  _hateneko_ok
  _hateneko_fix
  _hateneko_ng
  _hateneko_deleted
  _hateneko_logs

操作:
  - OKへ送る
  - Fixへ送る
  - NGへ送る
  - Deleteへ送る
  - ゴミ箱へ送る

初期設定ではDelete操作は完全削除ではなく、_hateneko_deleted への移動とする。
オプションでWindowsのゴミ箱へ送る設定を選べるようにする。
ゴミ箱送りには send2trash を使う。

ファイル移動時、同名ファイルが移動先にある場合は上書きしない。
例:
  image.png
  image_001.png
  image_002.png
のようにリネームして衝突回避する。


4.5 ショートカット

以下を実装する。

  ← または A: 前の画像
  → または D: 次の画像
  1: OKへ送る
  2: Fixへ送る
  3: NGへ送る
  Delete: Deleteへ送る
  Ctrl + Z: 直前の操作を戻す
  Space: 全体表示 / 拡大表示の切り替え
  F5: フォルダ再読み込み

仕分け後は設定に応じて自動的に次の画像へ進む。


4.6 Undo機能

直前の移動操作を取り消せるようにする。
最低限、1手戻しに対応する。
可能であれば履歴スタックで複数回Undoできるようにする。

Undo対象:
  - OKへ移動
  - Fixへ移動
  - NGへ移動
  - Deletedへ移動

ゴミ箱送りはUndo対象外でもよい。
その場合、UI上で警告する。


4.7 簡易スキャン機能

「スキャン開始」ボタンで、フォルダ内画像をチェックし、怪しい画像を抽出する。

初期MVPのチェック項目:

A. ファイル異常
  - 画像として開けない
  - ファイルサイズが0に近い
  - 読み込みエラーが発生する

B. 解像度異常
  - 設定した基準解像度と異なる画像を検出する
  - 例: 基準 1024 x 1536 に対して、512 x 768 や 1024 x 1024 などを警告

C. 縦横比異常
  - 基準縦横比と大きく異なる画像を検出する
  - 許容誤差は設定で変更可能にする

D. 真っ黒 / 真っ白 / 単色に近い画像
  - 平均輝度と分散を見て判定する
  - ほぼ単色で生成失敗っぽい画像を検出する

E. 重複画像
  - 初期版では完全一致ハッシュでよい
  - 可能であれば imagehash による pHash / average hash で近似重複も検出する

F. 顔数チェック
  - 初期版ではOpenCV等の簡易顔検出でよい
  - 1人絵設定なのに顔が2つ以上ある場合にSuspicious
  - 顔が0個の場合も設定によってSuspicious

------------------------------------------------------------
5. 将来追加したい破綻候補検出
------------------------------------------------------------

以下はMVP後の拡張として、検出器を追加できる構造にしておく。

5.1 頭・顔の数チェック

目的:
  - 1人絵なのに頭や顔が複数ある画像を検出する
  - 顔が多すぎる画像を抽出する

判定例:
  - expected_person_count = 1 のとき face_count >= 2 なら Suspicious
  - face_count = 0 かつ人物らしき領域ありなら Suspicious


5.2 腕の本数チェック

MediaPipe Pose等で肩・肘・手首のランドマークを取得し、腕の接続を確認する。

チェック対象:
  - 左肩、左肘、左手首
  - 右肩、右肘、右手首

判定例:
  - 肩から手首までの距離が極端に長い
  - 片側に複数の手首候補がある
  - 肩ではない位置から腕が生えているように見える
  - 手首が胴体内部に入り込んでいる


5.3 不自然な角度チェック

肩、肘、手首の3点から肘角度を計算する。

判定例:
  - 肘角度が極端に小さい
  - 肘が人体として不自然な方向に曲がっている
  - 左右の腕長が大きく異なる
  - 腕が胴体を貫通しているように見える

注意:
  ポーズやパースで誤検出が起こるため、「NG」ではなく「要確認」として扱う。


5.4 手・指チェック

MediaPipe Hands等で手のランドマークを取得する。

チェック対象:
  - 手の数
  - 指先候補数
  - 指の長さ
  - 指の並び
  - 手の位置
  - 手首との接続

判定例:
  - 1人絵で手が3つ以上検出される
  - 指先候補が6本以上
  - 指先候補が極端に密集している
  - 指の向きが大きく破綻している
  - 手が腕と接続していない

注意:
  握り手、隠れた手、小さい手、袖に隠れた手では誤検出が起こる。
  必ず「破綻」ではなく「破綻候補」として表示する。

7. スキャン結果データ形式
------------------------------------------------------------

画像ごとに以下のような結果を持つ。

例:

{
  "file_path": "C:/images/sample001.png",
  "status": "suspicious",
  "score": 72,
  "issues": [
    {
      "type": "resolution_mismatch",
      "severity": "warning",
      "message": "基準解像度 1024x1536 と異なります",
      "bbox": null
    },
    {
      "type": "too_many_faces",
      "severity": "danger",
      "message": "1人絵設定ですが顔が2つ検出されました",
      "bbox": [120, 80, 300, 260]
    }
  ]
}

bbox がある場合は画像上に赤枠や赤丸で表示する。

11. 検出器設計
------------------------------------------------------------

各検出器は共通インターフェースにする。

例:

class BaseDetector:
    name: str

    def detect(self, image, file_path: str, context: dict) -> list:
        raise NotImplementedError

戻り値は Issue のリストにする。

@dataclass
class Issue:
    type: str
    severity: str
    message: str
    bbox: tuple[int, int, int, int] | None = None
    score: float | None = None

Scannerは複数のDetectorを順番に実行し、結果を統合する。

class Scanner:
    def __init__(self, detectors: list[BaseDetector]):
        self.detectors = detectors

    def scan_image(self, file_path: str) -> ScanResult:
        ...

------------------------------------------------------------
14. 注意事項
------------------------------------------------------------

- 初期版では破綻判定の精度を追いすぎない。
- 「破綻」と断定せず、「破綻候補」「要確認」と表示する。
- 削除は完全削除にしない。
- 画像ファイル移動時は同名ファイル衝突に対応する。
- 大量画像でもUIが固まらないよう、スキャンは別スレッドで実行する。
- 画像読み込み失敗時にアプリが落ちないようにする。
- パスに日本語が含まれても動作するようにする。
- 将来の検出器追加を前提に、detectors配下を疎結合にする。
- まずは実用性優先。デザインはシンプルでよい。

------------------------------------------------------------
15. README用説明文
------------------------------------------------------------

破綻ねこは、Stable Diffusionなどで生成した画像の破綻候補を見つけ、ローカルで素早く仕分けるためのWindows向け検品アプリです。

指や腕、頭、顔、解像度違い、読み込みエラーなど、公開前に確認したい画像を抽出し、OK / Fix / NG / Deleteへ手早く分類できます。

初期版では、明確に検出しやすい異常候補を中心に扱います。
検出結果はあくまで確認補助であり、最終判断はユーザーが行います。
